# Art Gallery Microservices

Pet-проект для изучения микросервисной архитектуры, Kafka, Docker и CI/CD.

## 🎯 Цель проекта

Создать масштабируемую платформу для размещения и продажи произведений искусства (картины, фото, видео). Проект создаётся для самообразования и портфолио, чтобы освоить современный стек разработки.

## 🧩 Ключевые задачи

- Изучить микросервисную архитектуру на практике
- Настроить асинхронное взаимодействие между сервисами через Kafka
- Освоить контейнеризацию с Docker и оркестрацию
- Внедрить CI/CD с GitHub Actions
- Получить опыт работы с высоконагруженными системами

## 🛠️ Стек технологий

| Компонент | Технология |
|-----------|------------|
| Backend | C#, .NET 8, ASP.NET Core |
| Брокер сообщений | Apache Kafka |
| База данных | PostgreSQL (каждый сервис имеет свою схему) |
| API Gateway | YARP (Yet Another Reverse Proxy) |
| Фронтенд | React + TypeScript |
| Контейнеризация | Docker, Docker Compose |
| CI/CD | GitHub Actions |
| Кеширование | Redis |
| Хранилище файлов | MinIO (S3-совместимое) |
| Мониторинг | Serilog + Seq, Prometheus + Grafana |

## 📁 Структура проекта

Проект организован как монорепозиторий:

/art-gallery-microservices/
├── /docs/ # Документация и ADR
├── /src/ # Исходный код
│ ├── /ApiGateway/ # YARP Gateway
│ ├── /Services/
│ │ ├── /Identity/ # Аутентификация и пользователи
│ │ ├── /Gallery/ # Управление картинами
│ │ ├── /Upload/ # Загрузка файлов
│ │ └── /Notifications/ # Уведомления
│ └── /Shared/ # Общие библиотеки
├── /tests/ # Интеграционные тесты
└── /deploy/ # Kubernetes манифесты (позже)


## 🚀 План реализации

1. **Этап 0:** Подготовка репозитория и документации (сделан)
2. **Этап 1:** Настройка Docker Compose с инфраструктурой (PostgreSQL, Kafka, Redis, MinIO)
3. **Этап 2:** Создание API Gateway и первого микросервиса (Gallery)
4. **Этап 3:** Добавление аутентификации (Identity Service + JWT)
5. **Этап 4:** Интеграция Kafka (асинхронная обработка загрузки картин)
6. **Этап 5:** Разработка фронтенда на React
7. **Этап 6:** Внедрение CI/CD через GitHub Actions
8. **Этап 7:** Деплой и мониторинг
